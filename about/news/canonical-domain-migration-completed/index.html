Canonical Domain Migration to phantombladezerowiki.wiki

The site has standardized its canonical origin as phantombladezerowiki.wiki after the earlier typo-domain issue.

What Is Confirmed

Canonical links, sitemap locations, structured data, hreflang targets, internal links, robots references, and generated page URLs use the corrected domain. The old domain should retain path-preserving 301 redirects so search engines can transfer historical signals. Both GSC properties should remain available during migration monitoring.

What It Means for Players

Future builds run a domain-residue check before packaging. The sitemap index is the single submission target, while old URLs should not appear as canonicals or new sitemap entries. Redirects, selected canonicals, indexed counts, and crawl errors should be monitored until Google consistently chooses the new domain.

Source and Verification

This update is based on Live canonical sitemap target. It separates confirmed statements from inferences and records the publication date so later announcements or retail testing can replace outdated information.

Next Update
